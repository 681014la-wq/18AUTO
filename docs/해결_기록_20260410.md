## 해결된 핵심 문제 3개

### 1. Slate.js 텍스트 입력 실패

**증상**: 프롬프트 입력 후 "입력 실패 — 재시도" 무한 반복, 또는 플레이스홀더 텍스트(14자)만 감지

**원인**: Content Script는 **ISOLATED world**에서 실행됨 → Slate.js(MAIN world)의 내부 상태를 절대 변경할 수 없음
- `document.execCommand('insertText')` → DOM만 변경, Slate re-render 시 덮어씀
- `ClipboardEvent` → clipboardData가 world 경계를 넘지 못함
- `nativeInputValueSetter` → TEXTAREA 전용, Slate contenteditable에 무효

**해결**:
```
service_worker.js → INJECT_TEXT 핸들러
  → chrome.scripting.executeScript({ world: 'MAIN' })
  → sel.selectAllChildren(el)  // 기존 텍스트 전체 선택
  → InputEvent('beforeinput', { inputType: 'insertText', data: text })
  → Slate가 preventDefault() 호출 → 내부 상태 정상 갱신
```

**핵심 코드** (service_worker.js):
```js
chrome.scripting.executeScript({
  target: { tabId },
  world: 'MAIN',
  func: (text) => {
    const el = document.querySelector('div[data-slate-editor="true"]');
    el.focus();
    const sel = window.getSelection();
    if (sel) sel.selectAllChildren(el);
    const handled = !el.dispatchEvent(new InputEvent('beforeinput', {
      bubbles: true, cancelable: true,
      inputType: 'insertText', data: text,
    }));
    return handled ? 'OK_BEFOREINPUT' : 'FAIL';
  },
  args: [msg.text]
});
```

### 2. 생성 버튼 못 찾음 (0개)

**증상**: `❌ 생성 버튼 없음 — 전체 버튼 목록(0개)`, 또는 잘못된 "add_2 만들기" 버튼 클릭

**원인**:
1. Google Flow가 **Shadow DOM / Material 컴포넌트** 사용 → `document.querySelectorAll('button')` = 0개
2. 텍스트 입력 후 "만들기" 버튼이 **아이콘 전용 화살표 버튼**으로 변경 → 텍스트 매칭 실패
3. "add_2 만들기" (추가 버튼)가 텍스트 매칭에 걸려 오클릭

**해결**:
```
service_worker.js → CLICK_GENERATE 핸들러
  → chrome.scripting.executeScript({ world: 'MAIN' })
  → Shadow DOM 재귀 탐색 (shadowRoot까지 수집)
  → 1순위: 입력창 근처 좌표 기반 가장 오른쪽+하단 버튼
  → 2순위: 텍스트 매칭 (만들기/생성/generate/create)
  → DENY: add_2, upload, 이미지 업로드, search, menu 등
```

**핵심 코드** (service_worker.js):
```js
// Shadow DOM 관통 버튼 수집
function collect(root) {
  root.querySelectorAll('button,[role="button"],[tabindex="0"]').forEach(b => allBtns.push(b));
  root.querySelectorAll('*').forEach(el => { if (el.shadowRoot) collect(el.shadowRoot); });
}
collect(document);

// 입력창 근처 가장 오른쪽+아래쪽 버튼 = 생성 버튼
nearby.sort((a, b) => {
  const ra = a.getBoundingClientRect(), rb = b.getBoundingClientRect();
  return (rb.right + rb.bottom) - (ra.right + ra.bottom);
});
nearby[0].click();
```

## 핵심 교훈

1. **Chrome 확장 Content Script = ISOLATED world** → 페이지 JS(React/Slate/Angular) 내부 상태 절대 변경 불가
2. **`chrome.scripting.executeScript({ world: 'MAIN' })`** 이 유일한 해결책
3. **`beforeinput` 이벤트** = Slate.js가 실제 리스닝하는 유일한 입력 이벤트
4. **`execCommand('insertText')`** 는 DOM만 변경 → Slate re-render 시 덮어씀
5. **Google Flow = Shadow DOM** → 일반 querySelectorAll 못 찾음 → 재귀 탐색 필수
6. **위치 기반 버튼 탐색** > 텍스트 매칭 (UI 상태에 따라 버튼 텍스트가 바뀜)

## 시도했지만 실패한 방법들

| 방법 | 결과 | 이유 |
|------|------|------|
| `execCommand('insertText')` (ISOLATED) | ❌ | Slate 내부 상태 갱신 안 됨 |
| `ClipboardEvent paste` (ISOLATED) | ❌ | clipboardData가 world 경계 못 넘음 |
| `nativeInputValueSetter` | ❌ | TEXTAREA 전용, contenteditable 무효 |
| `execCommand('insertText')` (MAIN) | ❌ | DOM 변경만, Slate re-render 덮어씀 |
| `ClipboardEvent paste` (MAIN) | △ | 동작하나 beforeinput보다 불안정 |
| `beforeinput` (MAIN) | ✅ | Slate가 직접 처리, 내부 상태 정상 갱신 |
